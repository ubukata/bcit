If you don not have the mongohacker in your CPU you can user the command .pretty() in the end and then mongodb will return the JSON with line breaks.

What if we want to restrict this query over a condition? We can use the parameters of the find function:

				
db.collection.find(query, fields)

Where query is like our WHERE function in a relational database and the fields are the attributes that we want to return.

So now we can write something like:

				
var p = db.pokemons.find(query)

And use the variable p Right? NO

The find command returns a CURSOR for the query, in other words, we have a special way to access the data in a CURSOR.

To actually have the object we can use the findOne command: 

				
var p = db.pokemons.findOne(query)

Now the object will be avaliable in the p variable anytime.

If you want to search for a string using case insensitivity we can use the following command after the string in the query:

				
var query = { name: /pikachu/i }

Update

There are two ways to update information using mongodb:

  • save
  • update

The save command insert or update an object, but for update we must get the object first, change it and update it using the command save. For the update command we don't need to get the entire object to change it.

The funcion update receives three parameters:

				
db.collection.update(query, mod, options);

Update Operators

There a many update operators avaliable in mongodb, in this section I will list a few of than but you can read more about it here.

$set

The set operator modify an attribute or create it case it doesn't exists. Using the Pokemon Testmon _id we can update again the values on it.

				
{ $set  : { filed : value } }

$unset

This command is the opposite of the $set. In this case unset will remove the attribute in your document. The syntax is this:

				
{ $unset : { field : 1} }

Example:

$push

It will also create or update an attribute that is of type Array. If the attribute exists and it is not an array it will throw an error.

				
{ $push : { filed : value } }

Example:

$pull

This command removes a value from a field if the field is type of Array.

				
{ $pull  : { filed : value } }

Example:

$addToSet

Similar to push it will add a value to an existing Array, the difference is that using addToSet the value that you want to insert will not be inserted if exists.

				
{ $addToSet : { filed : value } }

Delete (Remove or Drop)

There are two ways to delete information, we can use the commands remove or drop. Both commands will delete data but they have different approaches in how they manage the collection.

drop

The function drop will delete completely the collection, deleting it from the database.

				
db.collection_name.drop()

remove

The remove function will delete the data in a collection. The collection will still exists in the database when we use remove.

				
var query = {name: \squirtle\i}
db.pokemons.remove(query)

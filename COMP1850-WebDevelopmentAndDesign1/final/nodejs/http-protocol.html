3. HTTP Protocol

The HTTP or Hypertext Transfer Protocol module is the most important for us since we are developing a node.js web server and its work with many functionalities of the HTTP protocol. It is also a native module, in other words, we do not need to install anything to use it.

To use it in node.js we just need a simple line of code:

				
					require(‘http’)

For each HTTP request the header of the request contains many information about it like the verb. When the serve receives it and process it the server respond with HTTP Status Code.

HTTP Methods/Verbs

There are many verbs that a client can use to send a request. The most commons ones are: GET, POST, PUT, DELETE, HEAD, TRACE, OPTIONS and much more.

GET

The GET method requests a representation of the specified resource. Requests using GET should only retrieve data and should have no other effect. A get request can return XML or JSON for example.

Here is one example of a GET HTTP request:

				
					GET / HTTP/1.1
					Host: google.com
					Connection: close
					User-Agent: Chrome/46.0.2490.86
					Accept-Encoding: gzip
					Accept-Charset: ISO-8859-1,UTF-8;q=0.7,*;q=0.7
					Cache-Control: no
					Accept-Language: de,en;q=0.7,en-us;q=0.3

POST

The POST method requests that the server accept the entity enclosed in the request as a new subordinate of the web resource identified by the URI. The data POSTed might be, for example, an annotation for existing resources; a message for a bulletin board, newsgroup, mailing list, or comment thread; a block of data that is the result of submitting a web form to a data-handling process; or an item to add to a database.

PUT

The PUT method requests that the enclosed entity be stored under the supplied URI. If the URI refers to an already existing resource, it is modified; if the URI does not point to an existing resource, then the server can create the resource with that URI.

DELETE

The DELETE method deletes the specified resource according to the URI.

HTTP Status Codes

The return codes of the HTTP are composed by 3 digits which follow a pattern to better understanding of the return.

1XX - Informational

They are use to inform that Request was received and continuing process.

2XX - Success

Means that the request was received with success. For example, when web pages are finishes to load the server responds with a 200 status code - OK.

				
					HTTP/1.1 200 OK
					Connection: keep-alive
					Content-Encoding: gzip
					Content-Type: text/html
					Date: Sun, 06 Dec 2015 01:07:17 GMT
					ETag: W/"55f9df1c-23f"
					Last-Modified: Wed, 16 Sep 2015 21:29:00 GMT
					Server: nginx
					Transfer-Encoding: chunked

3XX - Redirect

With this code we can warn directly the HTTP header to redirect saving the JavaScript to do that for example.

4XX - Client Error

The 4xx class of status code is intended for situations in which the client seems to have erred. For example, the code 401 an Unauthorized status code, in order words, the user probably is not authenticated. The 404 is another common status code for Not Found.

5XX - Server Error

The server failed to fulfill an apparently valid request. The 500 is used for Internal Server Error and it is a generic error message, given when an unexpected condition was encountered and no more specific message is suitable.

You can find a list of all HTTP status codes here.
